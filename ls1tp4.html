ASEMBLER

    The assembler takes assembly code and transforms it into object code.

  • Although object code is mostly composed of machine code, it cannot be executed by the operating system.
  • Object code does not have the necessary references to external functions and libraries to properly operate.

    • The Assembler
    The assembler creates object code. On a UNIX system you may see files with a .o suffix (.OBJ on MSDOS) to indicate object code files. It is a program that takes basic computer instructions and converts them into a pattern of bits. The assembler program takes each program statement in the source program and generates a corresponding bit stream or pattern. The output of the assembler program is called the object code. The sequence of 0’s and 1’s that constitutes the object program is sometimes called machine code.
